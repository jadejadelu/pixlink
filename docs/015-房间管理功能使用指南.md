# 房间管理功能使用指南

## 1. 功能介绍

房间管理功能是 PixLink 系统的核心功能之一，允许用户创建、加入和管理虚拟房间，用于：

- 团队协作和通信
- 游戏共享和多人游戏
- 安全的网络隧道管理
- 实时消息传递

## 2. 使用流程

### 2.1 创建房间

1. **登录系统**：使用您的 PixLink 账号登录系统
2. **进入仪表板**：登录成功后，您将看到系统仪表板
3. **查看房间**：点击仪表板中的「查看房间」按钮，进入房间管理页面
4. **创建房间**：点击「创建房间」按钮，填写房间名称和选择可见性（公开/私有）
5. **确认创建**：点击「创建房间」按钮，系统将为您创建新房间

### 2.2 加入房间

1. **登录系统**：使用您的 PixLink 账号登录系统
2. **进入仪表板**：登录成功后，您将看到系统仪表板
3. **查看房间**：点击仪表板中的「查看房间」按钮，进入房间管理页面
4. **加入房间**：点击「加入房间」按钮，输入房间 ID（如果需要，输入邀请码）
5. **确认加入**：点击「加入房间」按钮，系统将把您添加到指定房间

### 2.3 管理房间

1. **查看房间详情**：在房间列表中，点击任意房间卡片，进入房间详情页面
2. **查看成员列表**：在房间详情页面，点击「成员」标签，查看房间所有成员
3. **发送消息**：在房间详情页面，点击「消息」标签，输入消息内容并发送
4. **管理隧道**：在房间详情页面，点击「隧道」标签，创建或管理网络隧道
5. **管理游戏共享**：在房间详情页面，点击「游戏共享」标签，创建或管理游戏共享
6. **离开房间**：在房间详情页面，点击「离开房间」按钮，退出当前房间
7. **删除房间**：如果您是房间房主，在房间详情页面，点击「删除房间」按钮，删除整个房间

## 5. 后端服务说明

### 5.5 游戏共享服务（GameShareService）

**功能**：处理游戏共享的创建、查询和删除

## 6. 数据库模型

### 6.1 Room 模型

**功能**：存储房间基本信息

**主要字段**：
- `id`：房间 ID
- `ownerId`：房主 ID
- `name`：房间名称
- `visibility`：房间可见性（PUBLIC/PRIVATE）
- `status`：房间状态（ACTIVE/INACTIVE/DISSOLVED）
- `createdAt`：创建时间
- `updatedAt`：更新时间

### 6.2 Membership 模型

**功能**：存储房间成员信息

**主要字段**：
- `id`：成员 ID
- `roomId`：房间 ID
- `userId`：用户 ID
- `role`：成员角色（OWNER/MEMBER）
- `joinedAt`：加入时间
- `state`：成员状态（ONLINE/OFFLINE）
- `lastSeen`：最后在线时间

### 6.3 Tunnel 模型

**功能**：存储网络隧道信息

**主要字段**：
- `id`：隧道 ID
- `roomId`：房间 ID
- `type`：隧道类型（TCP/UDP）
- `port`：端口号
- `mode`：隧道模式（P2P/RELAY）
- `state`：隧道状态（CREATED/CONNECTING/CONNECTED/DISCONNECTED/FAILED）
- `createdAt`：创建时间
- `updatedAt`：更新时间

### 6.4 Message 模型

**功能**：存储房间消息

**主要字段**：
- `id`：消息 ID
- `roomId`：房间 ID
- `userId`：发送者 ID
- `content`：消息内容
- `type`：消息类型（TEXT/SYSTEM）
- `createdAt`：创建时间

### 6.5 GameShare 模型

**功能**：存储游戏共享信息

**主要字段**：
- `id`：游戏共享 ID
- `roomId`：房间 ID
- `userId`：创建者 ID
- `title`：游戏标题
- `proto`：协议类型（TCP/UDP）
- `port`：端口号
- `templateKey`：模板键
- `createdAt`：创建时间

## 7. 常见问题与解决方案

### 7.1 无法创建房间

**问题**：点击「创建房间」按钮后，显示创建失败

**解决方案**：
- 检查房间名称是否为空
- 检查网络连接是否正常
- 检查服务端日志，查看具体错误信息

### 7.2 无法加入房间

**问题**：输入房间 ID 后，显示加入失败

**解决方案**：
- 检查房间 ID 是否正确
- 检查是否需要邀请码
- 检查网络连接是否正常
- 检查服务端日志，查看具体错误信息

### 7.3 无法发送消息

**问题**：在房间详情页面，输入消息后发送失败

**解决方案**：
- 检查消息内容是否为空
- 检查网络连接是否正常
- 检查是否仍然是房间成员
- 检查服务端日志，查看具体错误信息

### 7.4 无法创建隧道

**问题**：在房间详情页面，点击「创建隧道」按钮后失败

**解决方案**：
- 检查隧道类型和端口号是否正确
- 检查网络连接是否正常
- 检查是否有足够的权限
- 检查服务端日志，查看具体错误信息

### 7.5 无法创建游戏共享

**问题**：在房间详情页面，点击「创建游戏共享」按钮后失败

**解决方案**：
- 检查游戏标题、协议类型和端口号是否正确
- 检查网络连接是否正常
- 检查是否有足够的权限
- 检查服务端日志，查看具体错误信息

## 8. 扩展与定制

### 8.1 添加新的游戏模板

要添加新的游戏模板，您需要修改 `src/services/gameShareService.ts` 文件中的 `getGameTemplates()` 方法，添加新的游戏模板信息。

### 8.2 扩展房间功能

要扩展房间功能，您可以：
- 修改 `src/services/roomService.ts` 文件，添加新的服务方法
- 修改 `src/controllers/roomController.ts` 文件，添加新的 API 端点
- 修改 `src/routes/roomRoutes.ts` 文件，注册新的路由
- 创建新的前端组件，实现新功能的用户界面

### 8.3 集成第三方服务

要集成第三方服务，您可以：
- 创建新的服务文件，实现与第三方服务的通信
- 在房间服务中调用第三方服务的 API
- 在前端组件中添加相应的用户界面
