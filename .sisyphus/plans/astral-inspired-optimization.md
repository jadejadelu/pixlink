# PixLink 设计优化方案（基于 Astral 启发）

**版本:** 2.0  
**日期:** 2026-02-24  
**参考:** Astral (基于 EasyTier 的游戏联机软件)

---

## 1. 核心洞察：Astral vs PixLink

### 1.1 本质差异

**Astral 的定位：**
- 工具型产品 - 专注"建立网络连接"
- 极简主义 - 最少步骤完成目标
- 网络层透明 - 用户直接使用虚拟 IP

**PixLink 的定位：**
- 社交型产品 - 专注"多人协作游戏"
- 流程化 - 准备 → 开始 → 游戏
- 应用层集成 - 内置聊天、状态同步

### 1.2 关键对比

| 场景 | Astral | PixLink | 优劣分析 |
|------|--------|---------|----------|
| **创建房间** | 3 步：填写信息 → 创建 → 连接 | 多步：创建 → 配置 → 邀请 | Astral 更快 |
| **加入房间** | 分享码/房间号，一键加入 | 邀请链接，需注册登录 | Astral 门槛低 |
| **开始游戏** | 用户手动使用虚拟 IP | 一键启动，自动连接 | PixLink 更自动化 |
| **沟通协调** | 无（依赖外部工具） | 内置聊天、准备系统 | PixLink 更完整 |
| **游戏集成** | 无（纯网络工具） | 计划集成游戏启动 | PixLink 更长尾 |

### 1.3 关键启示

**Astral 的成功要素：**
1. **零门槛加入** - 无需注册即可加入房间（分享码机制）
2. **房间号系统** - 类似 QQ 群号，便于口头分享
3. **极简化连接** - 选中 → 连接，两步完成
4. **透明化网络** - 虚拟 IP 直接展示，用户可自行使用

**PixLink 的差异化机会：**
1. **全程自动化** - 不只是建网，而是"一键联机"
2. **游戏内协调** - 准备状态、聊天、游戏状态同步
3. **社交沉淀** - 好友系统、历史房间、游戏统计
4. **桌面端深度集成** - Tauri 可实现的游戏进程交互

## 2. 优化方案

### 2.1 新增：房间号系统（降低分享门槛）

#### 问题
当前 PixLink 只能通过邀请链接分享，需要：
- 发送链接 → 朋友点击 → 注册/登录 → 加入

Astral 的房间号机制：
- 告诉朋友"房间号 9527" → 输入 → 直接加入

#### 解决方案

**新增房间号（Room ID）系统：**

```typescript
// 房间数据结构
interface Room {
  id: string;           // 内部 UUID
  roomNumber: string;   // 对外展示的房间号（如 "9527"）
  name: string;
  password?: string;    // 可选密码
  // ...
}
```

**房间号生成规则：**
- 4-6 位数字（易记忆）
- 可自定义（类似 QQ 群号）
- 唯一性校验

### 2.2 优化：连接流程简化

#### 问题
当前流程：创建房间 → 邀请朋友 → 朋友准备 → 房主开始游戏

过于复杂，特别是"准备"环节增加了认知负担。

#### 解决方案：两种模式并存

**模式 A：快速模式（新增，类似 Astral）**
- 适合：朋友间快速联机
- 流程：加入 → 自动连接 → 显示虚拟 IP → 手动游戏中输入

**模式 B：引导模式（保留）**
- 适合：新手、需要协调的情况
- 流程：准备 → 开始 → 自动启动游戏 → 自动连接

**UI 实现：**

房间设置中新增选项：
```
连接模式：
○ 快速模式（推荐）- 自动连接，显示虚拟 IP
  适合：熟悉游戏的玩家

● 引导模式 - 准备确认，一键启动
  适合：新手、需要协调的场合
```

### 2.3 优化：首次用户体验（FTUE）

#### Astral 的 FTUE
1. 下载安装
2. 基础设置（用户名、服务器）
3. 创建/加入房间
4. 点击连接
5. **直接使用虚拟 IP**

#### PixLink 的优化 FTUE

**第 0 步：零注册体验（可选）**

允许游客模式加入房间：
```

```

**第 1 步：简化设备配置**

当前痛点：生成身份 → 上传 → 接收 Permit → 导入（4 步）

优化方案：

## 5. 设计原则总结（借鉴 Astral）

### 5.1 极简主义

**Astral 理念：** "几分钟即可完成配置"

**PixLink 实践：**
- 默认值智能设置，减少用户选择
- 后台处理复杂流程，前台显示进度
- 提供"快速模式"和"引导模式"两种选择

### 5.2 透明化

**Astral 理念：** 虚拟 IP 直接展示，用户可自行使用

**PixLink 实践：**
- 同时提供"自动化"和"手动"两种选择
- 技术信息（IP、延迟、NAT）清晰展示
- 网络状态实时可见

### 5.3 低门槛分享

**Astral 理念：** 房间号 + 分享码，无需复杂操作

**PixLink 实践：**
- 保留链接分享（适合微信/QQ）
- 新增房间号（适合口头分享）
- 新增分享码（适合一键分享）
- 新增二维码（适合面对面）

### 5.4 渐进式复杂度

**新手：** 引导模式，每一步都有提示
**老手：** 快速模式，两步完成连接
**专家：** 显示高级信息，完全手动控制

## 6. 竞品差异化定位

### Astral
- **定位：** 工具型网络连接软件
- **优势：** 极简、跨平台、轻量
- **劣势：** 缺乏社交功能、游戏集成

### PixLink（优化后）
- **定位：** 社交化游戏联机平台
- **优势：** 一键联机、内置协调、游戏集成、桌面深度集成
- **劣势：** 相对复杂（但提供快速模式平衡）

**核心差异化：**
> "Astral 帮你建立网络，PixLink 帮你完成游戏"
