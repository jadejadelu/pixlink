# PixLink 错误码手册

**版本:** 1.0  
**日期:** 2026-02-24  
**适用范围:** REST API / WebSocket / 客户端错误

## 1. 错误码格式

**格式:** `CATEGORY_NUMBER`

**分类前缀:**
| 前缀 | 含义 | 示例 |
|------|------|------|
| `AUTH` | 认证相关 | `AUTH_001` |
| `ROOM` | 房间相关 | `ROOM_001` |
| `USER` | 用户相关 | `USER_001` |
| `DEVICE` | 设备相关 | `DEVICE_001` |
| `CONN` | 连接相关 | `CONN_001` |
| `SHARE` | 分享码相关 | `SHARE_001` |
| `ZTM` | ZTM 网络相关 | `ZTM_001` |
| `WS` | WebSocket 相关 | `WS_001` |
| `VALID` | 参数校验 | `VALID_001` |
| `RATE` | 速率限制 | `RATE_001` |
| `SYS` | 系统错误 | `SYS_001` |

## 13. 错误处理最佳实践

### 13.1 服务端实现

**统一错误响应:**
```typescript
// middleware/errorHandler.ts
export function errorHandler(err: Error, req: Request, res: Response, next: NextFunction) {
  if (err instanceof AppError) {
    return res.status(err.statusCode).json({
      success: false,
      error: err.message,
      code: err.code,
      details: err.details
    });
  }
  
  // 未知错误
  logger.error('Unhandled error:', err);
  res.status(500).json({
    success: false,
    error: '服务器内部错误',
    code: 'SYS_001'
  });
}

// AppError 类
export class AppError extends Error {
  constructor(
    message: string,
    public statusCode: number,
    public code: string,
    public details?: any
  ) {
    super(message);
  }
}

// 使用示例
throw new AppError(
  '房间不存在',
  404,
  'ROOM_003',
  { roomNumber: '9527' }
);
```

### 13.2 客户端实现

**错误处理工具:**
```typescript
// utils/errorHandler.ts
const errorMessages: Record<string, string> = {
  'AUTH_001': '登录已过期，请重新登录',
  'ROOM_003': '房间不存在或已解散',
  'CONN_001': '连接超时，请检查网络',
  // ...
};

export function handleApiError(error: ApiError): void {
  const userMessage = errorMessages[error.code] || error.message || '操作失败';
  
  // 根据错误类型处理
  switch (error.code?.split('_')[0]) {
    case 'AUTH':
      // 认证错误，跳转登录
      if (error.code === 'AUTH_001') {
        router.push('/login');
      }
      break;
      
    case 'ROOM':
      // 房间错误，显示提示
      toast.error(userMessage);
      break;
      
    case 'RATE':
      // 速率限制，显示倒计时
      showRetryCountdown(error.retryAfter);
      break;
      
    default:
      toast.error(userMessage);
  }
}
```

### 13.3 错误上报

**客户端错误上报:**
```typescript
// 上报严重错误
function reportError(error: Error, context: any) {
  if (process.env.NODE_ENV === 'production') {
    fetch('/api/client-errors', {
      method: 'POST',
      body: JSON.stringify({
        error: error.message,
        stack: error.stack,
        context,
        userAgent: navigator.userAgent,
        timestamp: Date.now()
      })
    });
  }
}
```

## 14. 文档结束

**使用建议:**
1. 服务端返回错误时务必包含错误码
2. 客户端根据错误码显示友好的用户提示
3. 记录错误日志以便排查问题
4. 敏感错误不要暴露详细信息给客户端

**维护:**
- 新增错误码时更新本文档
- 定期审查错误码使用情况
- 收集用户反馈优化错误提示
