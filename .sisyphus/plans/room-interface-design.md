# PixLink 房间内界面详细设计

**版本:** 1.0  
**日期:** 2026-02-24  
**设计目标:** 打造流畅的多人联机协调体验

---

## 1. 设计目标与原则

### 1.1 核心目标

1. **实时感知** - 用户能一眼看到谁在线、谁准备好了、连接状态如何
2. **高效协调** - 一键准备、快捷消息、游戏状态同步
3. **低门槛** - 非技术用户也能理解连接状态
4. **沉浸式** - 减少界面干扰，让用户专注于游戏准备

### 1.2 设计原则

- **信息层级清晰** - 最重要的信息（谁在线、连接状态）最显眼
- **操作就近** - 聊天输入框靠近聊天历史，准备按钮靠近玩家列表
- **状态可视化** - 用颜色、图标、动画传达状态，不只是文字
- **容错友好** - 连接失败时提供清晰的解决步骤

## 2. 界面布局详解

### 2.1 整体布局（三栏响应式）

```
┌──────────────────────────────────────────────────────────────┐
│  🎮 房间名称                              [设置] [分享] [✕] │ ← 顶部栏
├───────────────┬──────────────────────────────┬───────────────┤
│               │                              │               │
│  左栏         │      中栏（主内容区）         │   右栏        │
│  玩家列表     │                              │   聊天区      │
│  300px        │      自适应（最小 400px）      │   280px       │
│               │                              │               │
│               │   [连接状态/游戏画面/        │               │
│               │    错误提示]                  │               │
│               │                              │               │
│               │                              │               │
│               │   [主要操作按钮]              │               │
│               │                              │               │
└───────────────┴──────────────────────────────┴───────────────┘
       ↓                    ↓                       ↓
   玩家管理           连接/游戏控制              沟通协调
```

**响应式规则：**
- 窗口宽度 < 1000px：右栏收起为图标，点击展开
- 窗口宽度 < 800px：左栏收缩为迷你模式（只显示头像+状态）
- 窗口高度 < 600px：顶部栏简化，隐藏房间描述

### 2.2 顶部栏设计

```
┌──────────────────────────────────────────────────────────────┐
│ 🎮 Minecraft 生存服                          [⚙️] [🔗] [✕]  │
│ 房主：老王 | 房间ID: #12345 | 🔒 私密房间                      │
└──────────────────────────────────────────────────────────────┘
```

**元素说明：**

| 元素 | 说明 | 交互 |
|------|------|------|
| 🎮 图标 | 游戏类型图标（根据房间设置） | 悬停显示游戏名 |
| 房间名称 | 最大 30 字符，超出截断显示... | 点击可编辑（房主） |
| 房间ID | 短链接ID，方便分享 | 点击复制到剪贴板 |
| 🔒/🌐 | 私密（锁）/公开（地球） | 悬停显示"私密/公开房间" |
| ⚙️ 设置 | 打开房间设置面板 | 下拉菜单 |
| 🔗 分享 | 复制邀请链接 | 点击后显示"已复制"提示 |
| ✕ 退出 | 离开房间 | 二次确认对话框 |

**房主设置下拉菜单：**
```
⚙️ 房间设置
├── 修改房间名称
├── 修改房间描述
├── 切换房间类型（公开/私密）
├── 设置最大人数
├── ─────────────
└── 解散房间 ⚠️
```

**身份标识：**
- 👑 房主（金色）

**状态设计：**

| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| 已准备 | ✓ | 绿色 | 随时可以开始 |
| 准备中 | ⏳ | 橙色 | 等待操作 |
| 未准备 | ○ | 灰色 | 刚加入或取消准备 |
| 游戏中 | 🎮 | 蓝色 | 正在运行游戏 |
| 连接中 | 🔄 | 黄色旋转 | 隧道建立中 |
| 离线 | ⚠️ | 红色 | 网络断开 |

**延迟（Ping）显示规则：**
- < 50ms：🟢 绿色 - 极佳
- 50-100ms：🟡 黄色 - 良好
- 100-200ms：🟠 橙色 - 可接受
- > 200ms：🔴 红色 - 较差

**房主操作：**
- **私聊** - 打开私聊窗口

- **禁言** - 禁止该玩家发言

## 3. 交互流程详解

### 3.1 房主启动游戏流程

```
1. 所有玩家准备完毕
   ↓
2. 房主按钮变为 [🎮 开始游戏]（绿色高亮）
   ↓
3. 房主点击
   ↓
4. 系统检查：
   a. 所有人连接状态 OK？
   b. 隧道建立 OK？
   c. 游戏可启动？
   ↓
5. 如果检查通过：
   - 显示倒计时 3-2-1
   - 自动启动房主的游戏
   - 给其他玩家发送"房主已启动"通知
   ↓
6. 玩家收到通知，点击 [加入游戏]
   ↓
7. 自动复制服务器地址到剪贴板
   ↓
8. 界面显示"在游戏中"状态
```

### 3.2 玩家加入流程

```
1. 收到邀请/点击房间
   ↓
2. 进入房间，显示"准备中"
   ↓
3. 玩家检查自己的：
   - 网络连接状态
   - 游戏是否已安装
   - MOD 是否匹配
   ↓
4. 点击 [✓ 准备]
   ↓
5. 系统：
   a. 建立隧道连接
   b. Ping 测试
   c. 显示延迟
   ↓
6. 状态变为 [✓ 已准备]
   ↓
7. 等待房主启动
   ↓
8. 收到启动通知，点击 [加入游戏]
```

### 3.3 连接失败重试流程

```
1. 连接建立失败
   ↓
2. 显示错误面板（见 4.2）
   ↓
3. 用户点击 [重试]
   ↓
4. 系统：
   a. 重置连接
   b. 重新建立隧道
   c. 最多重试 3 次
   ↓
5. 如果仍失败，显示详细故障排除
```

## 4. 错误处理设计

### 4.1 错误类型分类

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| **E1 网络错误** | 连接断开、超时 | 自动重试 + 手动重试 |
| **E2 配置错误** | 端口冲突、权限不足 | 显示解决步骤 |
| **E3 用户错误** | 未安装游戏、版本不匹配 | 引导下载/更新 |
| **E4 系统错误** | 内部错误、未知异常 | 显示错误码 + 报告按钮 |

## 5. 特殊场景设计

### 5.1 断线重连

**场景：** 玩家网络波动，短暂断开

```
1. 检测到断开
   ↓
2. 显示重连提示：
   "网络已断开，正在尝试重连..."
   [重连中...] [退出房间]
   ↓
3. 自动重试（最多 5 次，间隔 2s）
   ↓
4a. 成功：
   - 显示"已恢复连接"
   - 恢复之前的状态
   - 同步错过的消息
   ↓
4b. 失败：
   - 显示"连接已丢失"
   - 提供 [重新加入] 按钮
```

**UI 表现：**
- 顶部栏显示黄色警告条
- 玩家状态变为"重连中"（旋转图标）
- 聊天显示"[系统] 小明 已离线，正在重连..."

### 5.2 房主掉线

**场景：** 房主意外退出

```
1. 检测到房主离线
   ↓
2. 系统消息："房主已离线，5分钟后自动解散房间"
   ↓
3. 倒计时显示在顶部栏
   ↓
4a. 房主在 5 分钟内重连：
   - 恢复正常
   ↓
4b. 房主未重连：
   - 自动转让房主给最早加入的玩家
   - 或解散房间（根据设置）
```

### 5.3 中途加入

**场景：** 游戏已开始，新玩家加入

```
1. 新玩家点击邀请链接
   ↓
2. 显示提示：
   "游戏已经开始，是否加入？"
   "当前 2 人正在游戏中"
   [加入游戏] [稍后加入]
   ↓
3. 新玩家点击加入
   ↓
4. 快速建立连接（跳过准备阶段）
   ↓
5. 直接显示已连接状态
   ↓
6. 提示：
   "您已加入！请打开游戏连接到：10.0.1.5:25565"
```

### 5.4 游戏崩溃处理

```
1. 检测到游戏进程消失
   ↓
2. 系统消息："检测到游戏已关闭"
   ↓
3. 玩家状态变为"未准备"
   ↓
4. 显示：
   "游戏似乎已崩溃或关闭"
   [重新启动游戏] [标记为已退出]
   ↓
5a. 点击重启：
   - 重新启动游戏
   - 自动重连隧道
   ↓
5b. 标记退出：
   - 状态变为观战/离开
   - 不占用玩家位
```

## 7. 数据与状态管理

### 7.1 Pinia Store 结构

```typescript
interface RoomState {
  // 房间信息
  room: {
    id: string;
    name: string;
    description: string;
    ownerId: string;
    isPrivate: boolean;
    maxPlayers: number;
    gameType: string;
    createdAt: Date;
  };
  
  // 玩家列表
  players: Player[];
  
  // 当前用户状态
  myStatus: 'waiting' | 'ready' | 'connecting' | 'connected' | 'gaming' | 'error';
  
  // 连接状态
  connection: {
    status: 'disconnected' | 'connecting' | 'connected' | 'error';
    virtualIp: string | null;
    port: number | null;
    ping: number;
    error: ConnectionError | null;
  };
  
  // 聊天消息
  messages: Message[];
  
  // 游戏状态
  gameSession: {
    isRunning: boolean;
    startTime: Date | null;
    playersInGame: string[];
  };
}
```

### 7.2 关键计算属性

```typescript
// 是否所有玩家都准备好了
const allPlayersReady = computed(() => 
  players.value.every(p => p.status === 'ready' || p.isOwner)
);

// 是否可以开始游戏（房主视角）
const canStartGame = computed(() => 
  isOwner.value && allPlayersReady.value 
  && connection.value.status === 'connected'
);

// 在线玩家数
const onlineCount = computed(() => 
  players.value.filter(p => p.status !== 'offline').length
);

// 未读消息数
const unreadCount = computed(() => 
  messages.value.filter(m => !m.read && m.senderId !== myId).length
);
```
